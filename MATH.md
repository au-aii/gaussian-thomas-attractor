## 元になる微分方程式

```
dx/dt = sin(y) − b·x
dy/dt = sin(z) − b·y
dz/dt = sin(x) − b·z
```

実装で触る数学は、実はこの3つの概念だけ。

---

## ① 常微分方程式（ODE）＝ 変化の速さのルール

- 左辺 `dx/dt` は「x が今この瞬間どれくらいの速さで動くか」。
- つまりこの式は **位置そのものではなく、位置の "変化率"（速度）** を決めている。
- → study plan の **微分積分（偏微分・勾配）** の本丸。

## ② オイラー法（数値積分）＝ コードのループの正体

微分方程式は「速度のルール」しかくれない。実際の軌道は自分で積み上げる：

```
次の位置 = 今の位置 + 速度 × dt
```

- code ではこの部分　`x_new = x + (sin(y) − b·x)·dt`
- **「速度を dt ぶんだけ足して、少しだけ進む」を数万回繰り返す** ＝ 積分を手で近似している。
- これがオイラー法。コードの for ループ1周＝時間を dt だけ進める1ステップ。

## ③ ベクトル場（vector field）＝ 空間の各点に刺さった矢印

```
F(x, y, z) = ( sin y − b·x ,  sin z − b·y ,  sin x − b·z )
```

- これは **空間のどの点にいても「次どっちに進むか」を返す関数**。
- 粒子は、その場所の矢印に沿って流れていくだけ。
- → study plan の **線形代数（ベクトル）** と **古典力学（運動方程式）** の交差点。

## マジックナンバーを説明できるようにする（理解の核）

| 記号         | 数学的な意味                                     | いじると何が起きる？（自分で実験して追記する）                                                            |
| ------------ | ------------------------------------------------ | --------------------------------------------------------------------------------------------------------- |
| `b = 0.1993` | **減衰項（摩擦）**。`−b·x` は速度を弱める力      | 大きく(例 0.5)→ 摩擦が勝って中心に収束し**形が消える**。`0.1993` は「カオスが持続するギリギリの弱い摩擦」 |
| `sin`        | **非線形・周期的な結合**。x→z→y→x と巡回的に絡む | これが "cyclically symmetric（巡回対称）" の語源。線形だと絡まず単純な渦になる                            |
| `dt = 0.05`  | **時間の刻み幅**。小さいほど積分が正確           | 大きすぎ(例 0.5)→ オイラー法の**離散化誤差**が爆発して軌道が発散。`≤0.05` 必須はこの誤差のため            |

> 🔑 README の「dt が大きいと発散する」という注意書きは、精神論ではなく
> **「オイラー法の誤差は dt に比例する」という数学的必然**として読める。

## セルフチェック（自分の言葉で言えればOK）

1. なぜ初期値を `(0,0,0)` にすると動かない？
   > `sin(0)=0`、全項ゼロ → 速度ゼロ → 永遠に止まる。
2. `dt` を 0.05 → 0.5 にしたら形は崩れる？
   > 崩れる。オイラー法の誤差が大きくなり、軌道が発散する。
3. `b` を 1.0 にしたら点群はどんな形になりそう？（予想 → 実験 → 答え合わせ）
   > 中心に収束する。減衰が強すぎてカオスが消える。
